OscarPredictions Tutorial

This tutorial turns the package workflow into a hands-on practice lab. You will install the package, initialize a workspace, generate features, run a model, and optionally test update/sync flows.

Prerequisites

  • Python 3.9+ installed.
  • Internet access for package installation (and for optional update/sync scraping steps).
  • A shell where you can run CLI commands.

If you are running this in Jupyter/Quarto, the cells below are executable as-is.

1) Set up helpers and a practice workspace

Run this first cell to define a helper for shell commands and create a local practice directory.

Code 
from __future__ import annotations

import subprocess
from pathlib import Path


def run(cmd: str, cwd: Path | None = None) -> None:
    print(f"$ {cmd}")
    result = subprocess.run(
        cmd,
        cwd=str(cwd) if cwd else None,
        shell=True,
        text=True,
        capture_output=True,
    )
    if result.stdout:
        print(result.stdout)
    if result.stderr:
        print(result.stderr)
    if result.returncode != 0:
        raise RuntimeError(f"Command failed with exit code {result.returncode}: {cmd}")

## Troubleshooting

- `oscar: command not found`: use `python3 -m oscar_predictions.cli --help` to verify install path, then reinstall in the active environment.
- Install fails on TestPyPI: retry with the exact index/extra-index flags shown above.
- Permissions/path issues: choose a writable workspace directory and rerun `init-data`.
- Slow or flaky update/sync runs: start with dry-run or capped scraping (`--max-movies`, `--max-actors`).
